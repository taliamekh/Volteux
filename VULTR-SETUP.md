# Vultr deployment guide for Kai

Generated by the deployment-scout agent. **Total time: ~75-90 minutes** for a first-timer. This is for Kai (he owns the backend); Talia handles Vercel for the frontend separately.

---

## 1. Sign up + claim MLH credits (~10 min)

Ask the **MLH coach at the hackathon for the Vultr Gift Code** — it's worth **$100 in credits, no credit card required**.

- Go to [mlh.com Vultr signup](https://www.mlh.com/challenges/019ce28a-00d0-9ec2-f07a-16ab8a608ac5)
- Create a Vultr account
- Redeem the gift code under **Billing → Promo Code**
- **Screenshot your Vultr profile landing page** — needed for the prize submission

**Gotcha:** if you sign up with a credit card first, you may not be eligible for the no-CC code. Use the gift code path.

---

## 2. Provision the VPS (~5 min)

Click **Deploy +** → **Cloud Compute** → **Regular Performance**.

- **Region:** pick one near the team (NY / Chicago / Toronto for North America) for low latency
- **OS:** Ubuntu 24.04 LTS
- **Plan:** **2 vCPU / 4GB RAM / 80GB SSD ≈ $24/mo** (~$0.036/hr — the $100 credit lasts ~115 days)
- **Hostname:** `volteux-pipeline`
- Skip backups / IPv6 / auto-backups for hackathon speed

Wait ~60 sec for boot. Copy the **public IPv4** and **root password** from the server's "Overview" tab.

---

## 3. SSH from Windows (~5 min)

Windows 11 PowerShell has SSH built in — skip PuTTY.

```powershell
ssh root@YOUR_PUBLIC_IP
```

Type `yes` to trust the host, paste the password (right-click pastes; nothing shows as you type — that's normal).

---

## 4. Install Bun + arduino-cli + AVR core (~15 min)

Paste these one block at a time inside the SSH session:

```bash
apt update && apt install -y unzip curl git
curl -fsSL https://bun.sh/install | bash
source ~/.bashrc
curl -fsSL https://raw.githubusercontent.com/arduino/arduino-cli/master/install.sh | sh
mv bin/arduino-cli /usr/local/bin/
arduino-cli core update-index
arduino-cli core install arduino:avr
```

Verify: `bun --version` and `arduino-cli version` should both print versions.

---

## 5. Clone the repo (~5 min)

```bash
cd /opt
git clone https://github.com/taliamekh/Volteux.git volteux
cd volteux
bun install
```

Then create `.env` with the Anthropic key:
```bash
nano .env
```
Paste:
```
ANTHROPIC_API_KEY=sk-ant-xxxxx
```
Save with **Ctrl+O**, Enter, **Ctrl+X**.

**Gotcha:** if the repo is private, generate a GitHub Personal Access Token and clone with `https://USER:TOKEN@github.com/...`.

---

## 6. Run the Hono server in the background (~5 min)

```bash
apt install -y tmux
tmux new -s pipeline
bun run start   # or whatever script Kai exposes
```

Press **Ctrl+B** then **D** to detach — the server keeps running after PowerShell closes. Re-attach later with `tmux attach -t pipeline`.

---

## 7. Open firewall + CORS (~10 min)

On the VPS:
```bash
ufw allow 22 && ufw allow 3000 && ufw --force enable
```
(Replace 3000 with whatever port Hono listens on.)

Also go to **Vultr dashboard → Firewall** and allow inbound TCP on that port — Vultr's network firewall is separate from `ufw`.

In Kai's Hono code, add CORS for the Vercel frontend:

```ts
import { cors } from 'hono/cors'
app.use('*', cors({ origin: 'https://your-app.vercel.app' }))
```

Test from a laptop:
```
curl http://YOUR_PUBLIC_IP:3000/health
```

---

## 8. (Optional, but probably needed) HTTPS via Caddy (~10 min)

**Critical gotcha:** if the Vercel frontend is served over HTTPS, the browser will block calls to a plain HTTP backend (mixed-content rule). Two fixes:

**A. Easy: install Caddy on the VPS for free auto-HTTPS.**
```bash
apt install -y caddy
```
Create `/etc/caddy/Caddyfile`:
```
your-domain.example.com {
  reverse_proxy localhost:3000
}
```
Caddy auto-fetches a Let's Encrypt cert. Need to point a domain's A record at the VPS first.

**B. Workaround for hackathon demo:** test with a Vercel preview URL (which can be configured to talk over HTTP), or skip Vercel and serve the frontend from the same Vultr box behind Caddy.

## Big gotchas — read these

1. **Don't commit `.env`** to the repo. The Anthropic API key must stay on the VPS.
2. **Mixed content blocking** — see Step 8.
3. **MLH gift code expires** — claim and redeem early.
4. **Anthropic API key never goes in the React frontend** — only on the backend.
