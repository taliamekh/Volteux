## Affected Pages/Routes

There is no production app yet. v0 ships a single-page SPA. The QA surfaces below are the routes/views the SPA will expose.

- `/` (home / single-page) — prompt input, classification result, four views (2D breadboard, 3D scene, sketch editor, parts list), flash button. Test that all four views render together for the canonical archetype-1 fixture.
- `/eval` (debug, internal) — read-only display of the latest eval run scores. Test that recent scores load and 4-axis scores are visible per case.

## Key Interactions to Verify

- **Prompt → classification**: type a free-form archetype-1-shaped prompt ("I want to detect when something gets close and turn a servo to wave at it"). Verify a confidence indicator appears, the classified archetype shows, and the four views render.
- **Out-of-scope prompt**: type a prompt that needs hardware not in the library ("I want to weigh things using a load cell"). Verify the Honest Gap message appears with the correct shape (`scope: out-of-scope`, `missing_capabilities`, beginner-readable `explanation`). Verify the cart and flash buttons are not shown.
- **3D component click**: click on a component body in the 3D view. Verify the `education_blurb` appears. Click on a labeled pin. Verify the pin's `description` appears.
- **2D breadboard render**: verify every component sits at its `anchor_hole` and that `rotation` properly rotates the rendered footprint. Verify all connections render with the correct `wire_color`. Verify wire colors match the rule "every connection to 5V/VCC is red; every connection to GND is black."
- **Cart button**: click "Buy parts." Verify a new tab opens with the Adafruit cart pre-filled with the correct SKUs and quantities (URL-inspect; do not need to actually purchase).
- **Flash button (Uno, real hardware)**: with a real Uno connected via USB, click "Flash to my Uno." Verify the WebUSB permission dialog appears, granting permission flashes the binary, and the Uno blinks/responds as expected.
- **URL share**: after generating a project, copy the URL. Open in a new incognito window. Verify all four views restore to the same state.

## Edge Cases

- **Empty prompt**: submit with no input. Verify a clear "type a description first" message; do not call the LLM.
- **Very long prompt** (>5000 chars): submit. Verify the prompt is rejected with a clear length-limit message OR the LLM call still works.
- **Rapid resubmit / double-click on Generate**: click Generate twice in 100ms. Verify only one LLM call is dispatched OR the second is queued/ignored. No duplicate views.
- **Network offline mid-generation**: disable network after the prompt is submitted but before the LLM responds. Verify the error boundary appears with the generic copy and a "Try again" button.
- **Compile API down**: simulate by pointing the frontend at a stopped backend. Verify error boundary on the flash button click; document hash still preserves the project.
- **WebUSB permission denied**: click flash, then click "Cancel" on the WebUSB picker. Verify the error boundary appears with the generic copy. Verify retry works (next click should re-trigger the picker).
- **No Uno connected**: click flash with no board. Verify a helpful message about connecting a board (case-specific is fine here since it's the most common error).
- **URL hash exceeds 2KB**: contrive a fixture that gzipped-base64-encodes to >2000 chars. Verify the app falls back to localStorage and shows the user a "your project is saved on this browser only" notice instead of a copyable URL.
- **localStorage disabled** (private browsing mode in some browsers): verify the app still functions in-memory; warn the user that the project won't persist across reloads.
- **Browser is not Chromium** (try in Safari or Firefox): verify a clear "this tool currently only works in Chrome or Edge" message before letting the user start.
- **Component count exceeds reasonable max** (e.g., LLM emits 10 components when archetype-1 expects 3-5): verify cross-consistency gate rejects with Honest Gap.
- **Stale URL** (project generated against schema v1, opened after schema v2 lands): verify a clear "this project was created in an older version" message; offer to regenerate.

## Critical Paths

These are the end-to-end flows that absolutely must work for the v0 demo:

1. **Beginner happy path** — type prompt → see classification → see all four views → click "Buy parts" → cart opens with correct SKUs → (parts arrive in real life) → click "Flash to my Uno" → Uno is flashed → blue light blinks (or whatever archetype-1 does).
2. **Beginner out-of-scope path** — type prompt that's outside archetype-1 → Honest Gap message appears with kind, beginner-readable copy → user knows what to try instead.
3. **3D learning path** — user generates a project, then clicks around the 3D view (without flashing). Each click reveals the correct `education_blurb` or `pin.description`. User leaves having learned what each component is.
4. **URL share path** — user shares the URL with a friend (in person or chat), friend opens it, sees the same project.
5. **Error recovery path** — at every error point, the user has a clear path back. Either retry, or fix input, or a "tell us what happened" link.

## Out of Scope for v0 QA

- ESP32-WROOM, ESP32-C3, Pi Pico flash flows (v1.5).
- Archetypes 2-5 (v1.5).
- Beauty pass on 3D models.
- AvantLink affiliate-link tracking.
- Mobile browser support (explicitly desktop Chrome/Edge only).
- WiFi / Adafruit IO setup flows.

## Hardware Required for v0 QA

- 1x Arduino Uno R3 with USB cable
- 1x HC-SR04 ultrasonic sensor
- 1x Micro servo SG90
- 1x breadboard
- jumper wires
- A Chrome or Edge browser on macOS, Windows, or Linux desktop.
